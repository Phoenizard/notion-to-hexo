# Contributing

Thank you for your interest in contributing to notion-to-hexo! We welcome bug reports, feature requests, and pull requests.

How to contribute
- Fork the repository and create a branch for your change.
- Make small, focused changes with clear commit messages (Conventional Commits are recommended).
- Run tests locally and include or update tests for new behavior.
- Open a pull request describing the change and linking any related issues.

Development setup
1. Create and activate a virtual environment (venv or conda).
2. Install dependencies:

```bash
pip install -r requirements.txt
```

Running tests

```bash
pytest
```

Coding style
- Keep code simple and readable. Follow existing project patterns.
- Prefer descriptive variable names; avoid single-letter names.

Maintainers will review PRs and may request changes before merge. For large or breaking changes, open an issue first to discuss the design.
